Live Server автоматически обновляет страницу в браузере при изменении HTML-файла, поэтому не нужно каждый раз вручную перезагружать страницу, как при обычном открытии через проводник. Это ускоряет разработку и позволяет сразу видеть изменения.

Тег `<main>` используется для обозначения основного содержимого страницы. Внутри него размещается главная информация сайта, которая отличается от меню, шапки (`header`) и подвала (`footer`).

Команда `git clone` нужна для копирования удалённого репозитория с GitHub на компьютер. Она создаёт локальную папку проекта вместе со всей историей изменений.

Атрибут `id` используется для уникальной идентификации HTML-элемента на странице. По `id` можно обращаться к элементу в CSS (`#id`) или JavaScript (`getElementById`).

Теги заголовков `<h1>`–`<h6>` отличаются уровнем важности и размером текста. `<h1>` — самый главный заголовок страницы, а `<h6>` — наименее важный и самый маленький.


1.`<link>` и `<script>`
<link>` подключает внешние файлы, чаще всего CSS. Ставится в `<head>` чтобы стили загрузились до того как пользователь увидит страницу.
<script>` подключает JavaScript. Принято ставить в конце `<body>` — перед `</body>`, чтобы сначала загрузился весь HTML и только потом JS начал с ним работать. Иначе скрипт попытается найти элементы которых ещё нет на странице.

2. Flexbox и направление
Flexbox активируется свойством:
display: flex;
Направление меняется через `flex-direction`. Внутри `@media` чтобы переключить с горизонтального на вертикальное пишут:
flex-direction: column;

3. Зачем нужен `@media`
Это условие которое говорит браузеру — "если экран меньше вот этого размера, применяй вот эти стили". Благодаря этому один и тот же сайт нормально выглядит и на большом мониторе и на телефоне. Без медиазапросов сайт на мобильном просто сжался бы или поехал.

4. Что такое DOM
Когда браузер загружает HTML — он превращает его в дерево объектов. Каждый тег становится объектом с которым можно работать через JavaScript. Это и есть DOM. Именно поэтому можно писать `document.getElementById` — ты обращаешься не к тексту HTML а к этому дереву объектов.

5. `document.getElementById()`
Ищет элемент по его `id` и возвращает его. Дальше с ним можно делать что угодно — менять текст, стили, вешать события.
Если передать несуществующий id — вернёт `null`. И если ты попытаешься что-то сделать с этим результатом, например `null.value` — получишь ошибку в консоли и скрипт сломается.

6. Что происходит при пустом поле
Срабатывает эта конструкция:
if (taskText === "") {
    // показываем ошибку
    return;
}
Это называется **условный оператор** `if`. Он проверяет условие — если поле пустое, показывает уведомление и выходит из функции через `return`. Всё что ниже не выполняется, задача не добавляется.
